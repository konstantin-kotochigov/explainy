# Промпты приложения

Эта директория содержит все промпты, используемые приложением для генерации объяснений технических тем.

## Структура

### Основная модель (Google Gemini)

Используется для генерации первоначального объяснения темы.

- **`main_system_prompt.txt`** - Системный промпт для основной модели, определяющий стиль и формат генерации объяснений
- **`main_user_prompt.txt`** - Шаблон пользовательского промпта для запроса объяснения темы
  - Параметры: `{topic}` - тема для объяснения

### Модель-критик (OpenAI GPT)

Используется для анализа и улучшения сгенерированного контента.

#### Критический анализ

- **`critic_system_prompt.txt`** - Системный промпт для модели-критика, определяющий требования к критике
- **`critic_user_prompt.txt`** - Шаблон промпта для генерации критического анализа
  - Параметры: 
    - `{topic}` - тема объяснения
    - `{content}` - содержимое для анализа

#### Генерация кода

- **`code_generation_system_prompt.txt`** - Системный промпт для генерации примеров кода
- **`code_generation_prompt.txt`** - Шаблон промпта для создания Python примеров
  - Параметры:
    - `{topic}` - тема объяснения
    - `{content}` - содержимое объяснения

## Использование параметров

Промпты содержат плейсхолдеры в фигурных скобках (например, `{topic}`, `{content}`), которые заменяются реальными значениями во время выполнения приложения методом `format()` или f-строками.

## Редактирование промптов

При изменении промптов учитывайте:

1. **Системные промпты** определяют общее поведение модели и должны оставаться стабильными
2. **Пользовательские промпты** могут быть адаптированы под конкретные задачи
3. **Параметры** не должны быть удалены из шаблонов, так как они используются в коде
4. **Формат Markdown** рекомендуется сохранять для корректного отображения в Jupyter Notebooks

## История

Промпты были реорганизованы в структурированную папку для улучшения читаемости и поддержки проекта. Ранее системные промпты хранились в корне проекта, а пользовательские промпты были встроены в код.
